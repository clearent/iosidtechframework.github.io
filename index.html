IDTech iOS Framework Overview

The IDTech iOS Framework gives your iOS App the ability to interact with an IDTech VP3300 Bluetooth credit card reader. The framework handles the credit card data coming from the reader on your behalf by sending it to our servers and returning a secured JWT (json web token) which you can present later to our mobile gateway endpoint for payment processing.

How it works

IDTech provides a framework for interacting with their reader. Clearent has wrapped the framework so it can handle the sensitive card data as well as provide a few extra methods for easier integration. The framework does not run the payment. It only secures the card data.

Recommended Implementation

The framework provides many capabilities but we have found the best implementation to be:

  1. Identify the reader first. With all new technologies the challenge is to get the User up to speed on its use. Even though the framework supports searching for the bluetooth reader and running the transaction in the same flow, we recommend separating out the two concerns so the experience at the time of running a transaction , while the customer is present, is optimized. In your app, provide a separate settings section allowing the User to set up the reader. Let them initiate a bluetooth search and select from one or more found readers. Then save the bluetooth info they selected.
  2. Display the status of the reader connection.
  3. Provide the User with an interface to start the transaction.
  4. Provide the User with an interface to cancel the transaction.
  5. Display messages to the User coming back from the framework (feedback callback).
  6. From your settings area provide the ability to force a bluetooth disconnect.

Bluetooth Search Flow

  1. Start a bluetooth search.
  2. Monitor for bluetooth devices sent back from framework.
  3. Communicate with User the device was found or if more than one found present a list for the User to choose from.
  4. Record the selected reader for future use.

Transaction Flow

  1. Start a transaction with a previously identified reader.
  2. Monitor for feedback from framework and display to User.
  3. Monitor for transaction token (JWT) from framework.
  4. Present transaction token to Clearent Mobile Gateway when ready to run a payment.
